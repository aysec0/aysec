Public API

Read-only HTTP API for aysec. Use it to build status pages, badge generators, scoreboard mirrors, notification bots — anything that needs platform data without scraping HTML.

Authentication

Every /api/v1/* request needs an Authorization: Bearer <token> header. Tokens look like aysec_pk_… and are created in Settings → API keys. They are read-only and tied to your account; revoke any time. 

curl https://aysec.me/api/v1/me \
  -H "Authorization: Bearer aysec_pk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

Rate limit

60 requests/minute per key. A bucket refills at 1 token/sec. When you exceed it you'll get 429 Too Many Requests with a Retry-After: 5 header.

Versioning

This is v1. Breaking changes (renames, removals, type changes) ship as /api/v2; additive changes (new fields, new endpoints) land in v1 in place. Build against the documented fields, ignore unknown ones.

Endpoints

GET /api/v1/me

Who's behind this token.

{
  "user": { "username": "ammar", "display_name": "Ammar", "avatar_url": null },
  "scopes": ["read"]
}

GET /api/v1/challenges

List published challenges. Query: ?category=web, ?limit=50, ?offset=0 (limit max 100).

{
  "challenges": [
    { "slug": "alg-none", "title": "alg=none", "category": "web", "difficulty": "easy", "points": 100, "solves": 47, "author": "ammar", "created_at": "2026-04-01 12:34:56" }
  ],
  "total": 23, "limit": 50, "offset": 0
}

GET /api/v1/challenges/:slug

Single challenge metadata + solve count. (No flag, obviously.)

GET /api/v1/leaderboard

Top 50 users by CTF score.

GET /api/v1/users/:username

Public profile: handle, avatar, bio, total solves, total score.

GET /api/v1/users/:username/solves

That user's solves, newest first. Query: ?limit=50 (max 100).

GET /api/v1/duels

Recent duels. Query: ?status=open|active|finished.

GET /api/v1/activity

Most recent 100 public activity events — feeds nicely into a status board.

Errors

JSON object, never bare strings:

{ "error": "Invalid or revoked token." }

Codes: 401 bad token, 404 unknown resource, 429 rate-limited, 500 server error.

Quick example

#!/usr/bin/env bash
TOKEN=aysec_pk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
curl -s https://aysec.me/api/v1/leaderboard \
  -H "Authorization: Bearer $TOKEN" \
  | jq '.leaderboard[0:3]'
